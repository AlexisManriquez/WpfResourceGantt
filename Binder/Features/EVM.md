# EVM

**Files**: `ProjectManagement/Features/EVM/EVMViewModel.cs` (~800 lines), `EVMView.xaml`

## Orchestration
- **Trigger**: `TacticalRibbonView` → EVM button
- **Command**: `MainViewModel.ShowEVMCommand`
- **Handler**: `MainViewModel.ShowEVMView()` → `CurrentView = "EVM"`

## Key Capabilities

### Zone A: Project Filter
- `ProjectOptions` combobox lists all Level 1 Projects across all systems
- Selection triggers `LoadDataForContext()` which generates chart + KPIs

### Zone B: KPI Cards
| Card | Formula | Source |
|------|---------|--------|
| SPI | `BCWP / BCWS` | `EVMViewModel.CurrentSPI` |
| CPI | `BCWP / ACWP` | `EVMViewModel.CurrentCPI` |
| SV | `BCWP - BCWS` | `EVMViewModel.CurrentScheduleVariance` |
| CV | `BCWP - ACWP` | `EVMViewModel.CurrentCostVariance` |
| EAC | `BAC / CPI` | `EVMViewModel.EAC` |
| VAC | `BAC - EAC` | `EVMViewModel.VAC` |
| TCPI | `(BAC - BCWP) / (BAC - ACWP)` | `EVMViewModel.TCPI` |

All KPI values are read from the pre-computed, persisted database values set by `EvmCalculationService` at load time.

### Display Modes
The application supports a global toggle between **Dollars** and **Hours** modes.

#### Core Values — change with mode
| DoD Term | Dollars Mode | Hours Mode (DAU "pure effort") |
|----------|-------------|----------------------------------|
| **BAC** | Total planned cost ($) | Total planned hours |
| **BCWS / PV** | Planned Value ($) | Planned Hours |
| **BCWP / EV** | Earned Value ($) | Earned Hours |
| **ACWP / AC** | Actual Cost ($) | Actual Hours (timesheets) |
| **SV** | $ ahead/behind schedule | Hours ahead/behind schedule |
| **CV** | $ under/over budget | *Labor Efficiency Variance* — hours saved/lost |
| **EAC** | Estimated total cost ($) | Estimated total hours |
| **VAC** | $ over/under at completion | Hours over/under at completion |

#### Indices — always unitless (same in both modes)
| Index | Formula | Why unitless |
|-------|---------|---------------|
| **SPI** | `EV ÷ PV` | Same-unit division cancels the unit |
| **CPI** | `EV ÷ AC` | Same-unit division cancels the unit |
| **TCPI** | `(BAC-EV) ÷ (BAC-AC)` | All terms in same unit |

> [!NOTE]
> When the mode is toggled, `DataService.ToggleEvmModeAsync()` scales BAC and ACWP on **leaf** `WorkBreakdownItem` nodes (÷195 for Hours, ×195 for Dollars). BCWS/BCWP are recalculated from the updated BAC by `EvmCalculationService` on next load. All KPI cards and the S-Curve recalculate in the correct unit automatically.

| Global Flag | Value |
|-------------|-------|
| `IsEvmHoursBased = true` | Hours mode active |
| `IsEvmHoursBased = false` | Dollars mode active |

### Zone C: S-Curve Chart
- Generated by `GenerateSCurve()` using weekly time points
- Three lines: BCWS (blue), BCWP (black), ACWP (red)
- Optional dashed forecast line (EAC path)
- Uses `LiveCharts` `SeriesCollection`
- Data source: `ProgressHistory` records on leaf `WorkBreakdownItem` nodes

### Zone D: WBS Detail Table
- Flattened tree view with expand/collapse
- Each row shows per-item SPI, CPI, BAC, BCWS, BCWP, ACWP
- Clicking a row drills into that item's S-Curve
- All ACWP values are **read-only** — sourced from SMTS imports only

### Time Range Support
Supports filtering: 30d, 60d, 90d, 180d, 1yr, 3yr, All

## Simulation Impact Tracking
In the **Temporal Sandbox**, EVM metrics are used to measure the "Cost of Reality." The **Impact Bar** compares Live metrics (from the database) against Simulated metrics in real-time. This allows PMs to see how a schedule slip or progress gain affects SV, CPI, and EAC before committing to a recovery plan.

## Refresh After Close Week
When `MainViewModel.ExecuteCloseWeekAsync()` completes, it calls `EVMViewModel.Refresh()` if the EVM view is active, which re-calls `LoadDataForContext()` to reload the chart with the newly captured snapshot.

## Chart Data Source
- BCWS: Time-proportional using business days (see [[EVM Calculation Rules]])
- BCWP: From `ProgressHistory` records (latest entry at or before each date)
- ACWP: From `ProgressHistory.ActualWork` — read from SMTS-imported values only
